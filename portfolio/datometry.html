Datometry qInsight — Designing the MVP for Cloud Data Warehouse Virtualization

I led UX/UI for Datometry qInsight, a platform that helps enterprises run applications built for on-premises data warehouses on major cloud data warehouses—without rewriting code or changing configurations. This case study covers the MVP foundation, early UI system, and the collaboration model that enabled delivery.

Category Enterprise · Data Platform
Role Design Lead (UX/UI)
Delivery MVP · UI Foundation

About

Datometry qInsight was positioned as a “replatforming layer” for data warehousing. The product leveraged virtualization principles to allow applications originally developed for on-prem data warehouses to run on major cloud data warehouses without requiring application code changes. That promise created an important UX challenge: the product needed to present complex infrastructure concepts in a way that felt concrete, trustworthy, and operable.

My Role

I led design within a cross-functional delivery team and partnered closely with stakeholders, product owners, engineering, and QA. My responsibilities included shaping the MVP information architecture, defining core interaction patterns, producing wireframes and prototypes, and delivering frontend-ready UI specs that supported rapid implementation.

Goals

  • Design an interface that makes cloud replatforming workflows understandable and operable for enterprise users.
  • Reduce perceived risk by clarifying system status, outcomes, and dependencies before users take action.
  • Enable rapid adoption of new database technologies while preserving the value of existing applications.

Constraints

The MVP needed to ship quickly and communicate a technically sophisticated value proposition to users who expect reliability. Every screen had to balance speed of use with precision: the UI could not be ambiguous about environments, compatibility, or operational impact.

Process

We worked in an Agile model with short feedback loops. Design and engineering moved in parallel: I created prototypes early, validated flows with stakeholders, and evolved the UI alongside implementation constraints. This ensured the product remained cohesive even as scope evolved toward MVP essentials.

Understanding Phase

  • Studied the product’s virtualization approach and how it mapped to user workflows.
  • Aligned with stakeholders and product owners on MVP scope and success measures.
  • Identified the high-risk points where the UI must prevent costly mistakes.

Agile Collaboration

I participated in sprint planning, daily stand-ups, and sprint reviews to keep design decisions synchronized with delivery. This cadence helped resolve ambiguities quickly—especially around edge cases, terminology, and system state representation.

MVP Design

The MVP focused on the minimum set of workflows needed to demonstrate value end-to-end. I produced wireframes and interactive prototypes to validate navigation, data hierarchy, and the “happy path” as well as key error states. The goal was to make complex operations feel predictable through consistent patterns and clear feedback.

User Interface Design

I established foundational UI patterns for dashboards, tables, and status surfaces—prioritizing scannability, safe actions, and clear system feedback. Because enterprise users operate with high stakes, the UI emphasized:

  • Strong information hierarchy for quick scanning.
  • Consistent table and filtering patterns for operational tasks.
  • Explicit status and health indicators for confidence and troubleshooting.

Iterative Feedback

We incorporated feedback continuously—from stakeholders and team members, and when available, from representative users. Iteration focused on reducing ambiguity, improving terminology, and tightening the relationship between “what the system knows” and “what the user can safely do next.”

Results

The outcome was an MVP UI that effectively demonstrated qInsight’s core promise and gave engineering a strong foundation for future features. By leading design within an Agile delivery model, we kept the UI aligned with the product vision while moving quickly. The resulting experience emphasized clarity, trust, and operational readiness—critical qualities for enterprise adoption.

What I’m Proud Of

  • Building an MVP information architecture that scaled with feature growth.
  • Designing UI patterns that made complex system behavior legible and actionable.
  • Maintaining alignment across stakeholders through prototypes and clear specs.
